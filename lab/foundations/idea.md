## F-1. 空間ってなんだろう ― 集合から距離空間へ

**問い**: なぜ数学者は「空間」という言葉を使うのか？ℝ² と ℝ³ の何が違うのか？

### 集合
- 要素の集まりにすぎない。「近さ」も「大きさ」も定義されていない。
- `{1, 2, 3}` も `{りんご, みかん}` も「集合」

### 距離空間（Metric Space）
「近さ」だけを定義した空間。距離関数 `d(x, y)` が満たすべき 3 条件：
1. `d(x, y) ≥ 0`、かつ `d(x, y) = 0 ⟺ x = y`（同一点判定）
2. `d(x, y) = d(y, x)`（対称性）
3. `d(x, z) ≤ d(x, y) + d(y, z)`（三角不等式）

**具体例で理解する**:
- ユークリッド距離：`d(x, y) = ‖x - y‖₂`（直線距離）
- マンハッタン距離：`d(x, y) = Σ|xᵢ - yᵢ|`（碁盤の目）
- 最大値距離：`d(x, y) = max|xᵢ - yᵢ|`（チェビシェフ）

→ 同じ「点の集まり」でも距離の定義次第で「形」が変わる。

**コード実験** `exp_F1.py`：
- ℝ² 上の同一点集合に 3 種の距離を定義し、「単位球」の形を可視化する
- ユークリッド球は丸、マンハッタン球は菱形、チェビシェフ球は正方形

## F-2. ノルム空間 ― 「大きさ」を定義する

**問い**: 距離があれば十分では？なぜ「ノルム」が別に必要なのか？

距離空間は「2点間の距離」は測れるが、「ベクトルの足し算・スカラー倍」との整合性がない。
ノルム空間は**ベクトル空間の構造**に「大きさ」を乗せた空間。

### ノルムの条件
1. `‖x‖ ≥ 0`、かつ `‖x‖ = 0 ⟺ x = 0`
2. `‖αx‖ = |α|‖x‖`（スケーリング）
3. `‖x + y‖ ≤ ‖x‖ + ‖y‖`（三角不等式）

### 主なノルムの比較
| ノルム | 定義 | 意味 |
|---|---|---|
| L¹ | `Σ|xᵢ|` | 絶対値の和 |
| L² | `√(Σxᵢ²)` | ユークリッド距離 |
| L^∞ | `max|xᵢ|` | **最大成分の絶対値** |

**なぜこの論文は L^∞ を使うのか**:
L² は「平均的な大きさ」しか制御できない。作用素合成で一部の成分だけが爆発しても L² は気づかない。
L^∞ は**すべての点・成分を同時に制御**する → 点ごとの安定性を保証できる。

**コード実験** `exp_F2.py`：
- 同一ベクトルを L¹, L², L^∞ で測って比較
- 「一つの成分だけ大きな」ベクトルを作り、各ノルムの反応の違いを観察

## F-3. バナッハ空間 ― 「完備性」という保険

**問い**: ノルム空間に何が足りないのか？

### 完備性とは
コーシー列（要素同士が限りなく近づく列）が**必ず空間内の点に収束する**という性質。

**反例で理解する**: 有理数 ℚ はノルム空間だが完備でない。
- `3, 3.1, 3.14, 3.141, ...`（π に近づく列）は ℚ 内で収束先がない。
- ℝ は完備：すべてのコーシー列が ℝ 内に収束先を持つ。

### バナッハ空間 = 完備なノルム空間

**なぜ重要か**: 最適化アルゴリズムは「収束先が存在する」ことを前提にしている。
完備でない空間では「収束しているように見えて、収束先が空間の外にある」という事態が起きる。

主なバナッハ空間：
- `ℝⁿ`（有限次元は常に完備）
- `L^∞([0,1])`：本質的有界関数のなす空間
- `W^{1,∞}`：**この論文の舞台**

### ヒルベルト空間との関係
バナッハ空間に「内積」を加えるとヒルベルト空間（例：L²）。
L² は内積があって扱いやすいが、点ごとの制御ができない。
→ この論文があえてバナッハ空間（W^{1,∞}）を選ぶ理由がここにある。

**コード実験** `exp_F3.py`：
- コーシー列を生成し、ℝ（完備）と ℚ（非完備）での違いをシミュレート
- W^{1,∞} の収束とL² の収束を並べて比較

## F-4. ソボレフ空間 W^{1,∞} ― 微分まで制御する

**問い**: L^∞ だけでは不十分なのか？

### 弱微分（Weak Derivative）
古典的な微分は「微分可能な関数」にしか定義できない。
弱微分は「微分不可能でも意味のある微分」を分布論の枠組みで定義する。

### W^{1,∞} の定義
```
W^{1,∞} = { f : ‖f‖_{L^∞} < ∞ かつ ‖f'‖_{L^∞} < ∞ }
```
つまり：
- 関数自体が有界（`‖f‖_{L^∞} < ∞`）
- **微分（勾配）も有界**（`‖f'‖_{L^∞} < ∞`）

### なぜ微分まで制御するのか
- Lipschitz 連続性 ⟺ `‖f'‖_{L^∞} < ∞`
- 勾配が有界 → 最適化が安定（勾配爆発が起きない）
- mollifier 畳み込みの収束保証に微分の有界性が必要（Corollary 1）

**コード実験** `exp_F4.py`：
- W^{1,∞} ノルムの計算実装
- L^∞ は有界だが W^{1,∞} には属さない関数の例（微分が爆発する関数）
- 「W^{1,∞} に属す」という条件を数値的に確認

## F-5. 群論ことはじめ ― 変換の代数

**問い**: 「群」「モノイド」とは何か？なぜ介入作用素がモノイドを成すのか？

### 群（Group）
集合 G と演算 · が以下を満たすとき群：
1. **閉性**: `a, b ∈ G → a·b ∈ G`
2. **結合律**: `(a·b)·c = a·(b·c)`
3. **単位元**: ある `e ∈ G` で `a·e = e·a = a`
4. **逆元**: すべての `a` に `a⁻¹ ∈ G` が存在して `a·a⁻¹ = e`

**身近な例**:
- 整数 ℤ と加法：群（逆元は `-n`）
- 正の実数と乗法：群（逆元は `1/x`）
- 正方行列と乗法：**群でない**（逆行列が存在しない行列がある）

### モノイド（Monoid）
群から「逆元の存在」を除いたもの。

```
モノイド = 閉性 + 結合律 + 単位元
```

**なぜ介入作用素はモノイドか**:
- 閉性：`E_a ∘ E_b` も `P → P` の写像（合成で空間の外に出ない）
- 結合律：関数合成は常に結合的
- 単位元：恒等写像 `id(p) = p`
- 逆元：一般に存在しない（介入は不可逆）→ **群でなくモノイド**

### 非可換性（Non-commutativity）
群・モノイドは `a·b = b·a`（可換律）を要求しない。
- 整数の加法：可換（`3+5 = 5+3`）→ 可換群（アーベル群）
- 行列の乗法：非可換（`AB ≠ BA` 一般に）
- 介入作用素：非可換（`E_A ∘ E_B ≠ E_B ∘ E_A`）→ **これが論文の核心**

**コード実験** `exp_F5.py`：
- 具体的な 2×2 行列で `AB ≠ BA` を確認
- 介入作用素 E_A, E_B で `E_A(E_B(p)) ≠ E_B(E_A(p))` を可視化
- 「交換子」`[A,B] = AB - BA` のノルムと結果の差の相関を確認

## F-6. 作用素論 ― 変換を関数として扱う

**問い**: 「作用素」は「関数」と何が違うのか？

### 作用素（Operator）
関数空間から関数空間への写像。`T: X → Y`（X, Y は関数空間）。

**有界作用素**:
```
‖T‖ = sup_{‖x‖≤1} ‖Tx‖ < ∞
```
入力を「有界に引き伸ばす」作用素。

### Lipschitz 連続性
```
‖T(x) - T(y)‖ ≤ L · ‖x - y‖
```
「入力の差に比例した出力の差」の上界が存在する。
→ W^{1,∞} への埋め込みにより、作用素合成全体が Lipschitz になる。

### 合成の Lipschitz 定数
`T₁` の定数 `L₁`、`T₂` の定数 `L₂` のとき：
```
L_{T₁∘T₂} ≤ L₁ · L₂
```
合成を繰り返すと定数が掛け算で増える → W^{1,∞} はこれを制御する枠組み。

**コード実験** `exp_F6.py`：
- 作用素ノルムの数値計算（パワー法）
- Lipschitz 定数の推定と合成での変化
- 有界 vs 非有界作用素の動作の違い

## F-7. Fréchet 微分 ― 関数空間での微分

**問い**: ℝ → ℝ の微分は知っている。では「関数空間 → ℝ」の微分とは何か？

### 方向微分（Gâteaux 微分）
```
D_h F(x) = lim_{t→0} [F(x + th) - F(x)] / t
```
特定の方向 `h` への微分。存在しても全微分とは限らない。

### Fréchet 微分
すべての方向に一様に近似できる線形作用素 `DF(x)` が存在するとき：
```
lim_{‖h‖→0} ‖F(x+h) - F(x) - DF(x)[h]‖ / ‖h‖ = 0
```
→ 「一階近似が全方向で成立する」= 真の意味での微分可能性。

**なぜ重要か**:
- 勾配降下法の理論的根拠は Fréchet 微分可能性
- mollifier 平滑化後の `J_ε` が Fréchet 微分可能 = 勾配が Lipschitz 連続（Layer 3）

**コード実験** `exp_F7.py`：
- 有限差分による Fréchet 微分の数値近似
- 微分可能・不可能な汎関数の例
- `J_ε` の数値 Fréchet 微分と解析勾配の比較

## F-8. Mollifier ― 滑らかな近似の数学

**問い**: 畳み込みで「滑らか」にできるのはなぜか？

### 近似単位元（Approximate Identity）
関数列 `{ρ_ε}` が近似単位元 ⟺:
1. `∫ ρ_ε(x) dx = 1`（規格化）
2. `ρ_ε ≥ 0`（非負）
3. `supp(ρ_ε) → {0}`（ε → 0 でサポートが原点に集中）

### 標準 Mollifier
```
ρ_ε(x) = (1/ε) ρ(x/ε),  ρ(x) ∝ exp(-1/(1-|x|²))  (|x| < 1)
```
- `C^∞` 関数（無限回微分可能）
- コンパクトサポート（有限区間の外はゼロ）

### 畳み込みによる平滑化
```
f_ε = f * ρ_ε
```
- `f` が L^∞ なら `f_ε` は `C^∞`
- `‖f_ε - f‖_{L^∞} → 0`（ε → 0 で均一収束）

**コード実験** `exp_F8.py`：
- 標準 Mollifier カーネルの可視化
- 不連続関数・ステップ関数を mollify して平滑化過程を観察
- 収束 `‖f_ε - f‖_{L^∞}` の ε 依存性を log-log プロット
